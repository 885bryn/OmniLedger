## Visual States

### History Row — No Variance (actual equals projected or actual_amount is null)

- Shows: Paid badge, event title, item link, Actual date card cell, Actual paid card cell.
- Does NOT show: Overpaid or Underpaid badge, projected reference lines.
- Date displayed: `actual_date` → `completed_at` (day portion) → `due_date` fallback chain.

### History Row — Variance Present (actual_amount differs from projected amount)

- Shows: Paid badge + Overpaid or Underpaid badge (inline, after Paid badge), plus projected reference lines in both card cells.
- Projected reference line format in "Actual date" cell: `— projected: [formatted due_date]`
- Projected reference line format in "Actual paid" cell: `— projected: [formatted projected amount]`
- Manual Override rows: show Manual Override badge, Paid badge, and variance badge if applicable — all three can coexist.

### Decision on projected reference visibility (Claude's Discretion)

Show projected reference lines ONLY on rows where a variance badge appears (i.e., where actual_amount differs from projected). Rationale: always-visible projected text would add noise to the majority of rows where actual equals projected; the reference context is only meaningful when there is a discrepancy to explain.

## Interaction Contract

### No new user-triggered interactions

Phase 40 is display-only. There are no new buttons, dialogs, or forms. All changes are to rendering logic (labels, badge display, date/amount source, grouping sort key) and the backend `financial-metrics.js` resolver.

### Accessibility

- Variance badges must include `aria-label` when inline text alone is ambiguous. Recommended: `aria-label="Overpaid by {{formattedDiff}}"` or `aria-label="Underpaid by {{formattedDiff}}"` if the amount diff is computed and available; otherwise the badge text itself is sufficient as it is uppercase visible text.
- Projected reference lines are supplementary — they must NOT carry `role="alert"` or any live region attribute.
- History row `<article>` elements retain their existing `data-event-row-id` and `data-history-highlighted` attributes for test selectors.
